    开发 uTools 插件调用 Claude Code CLI 的踩坑记录

    概述

    最近想在 uTools 里直接问 Claude 问题，不需要打开终端、不需要切窗口，输入完直接出结果。思路很简单：uTools 超级面板输入问题，调用本地 Claude Code CLI，拿到回答后展示在插件窗口里。

    听起来不复杂，实际写的时候踩了不少坑。主要是 uTools 基于 Electron，插件分两层上下文，加上 Claude Code 本身是个 TUI 工具，非交互调用很容易出问题。这篇文章把整个过程和踩的坑记下来，方便以后参考。

    插件目录结构

    uTools 插件本质上是一个文件夹，里面包含：

    plugin.json 定义插件名称、入口文件和 feature 列表。preload.js 跑在 Node.js 上下文，可以调 child_process 这些原生模块。index.html 跑在沙盒浏览器里，两者通过 Electron 的 contextBridge（或直接挂载 window）通信。

    uTools 插件的两层上下文

    这是第一个要理解清楚的地方。uTools 基于 Electron，插件代码分两个环境跑：

    层级 文件 权限
    Preload preload.js 可以 require('child_process')，有完整系统权限
    Renderer index.html 只能访问 Web API + Preload 暴露出来的接口

    preload.js 通过 window.exports 注册 feature 的生命周期回调（enter / search / select），同时在 window 上挂载供前端调用的 API。前端调用后，实际执行系统命令的逻辑都在 preload.js 里完成。

    踩坑三：Claude Code CLI 的非交互调用

    最初以为像普通 CLI 一样传个 --stdin 就行：

    结果进程直接挂了。Claude Code 本质上是 TUI 工具（基于 blessed 这类库），在非 TTY 子进程里行为很不稳定。而且 --stdin 这个参数根本不存在。

    解决思路：

    • 去掉不确定的参数，直接 spawn('claude', [], ...)
    • 通过 child.stdin.write(question + '\n') 把问题写进去
    • 延迟几秒再写 /exit\n 结束会话
    • 加上环境变量 CI: '1' 尝试让它别进 TUI 模式

    另外加了个预检，插件启动时执行 claude --version，把结果通过 claudeStatus 暴露给前端，方便用户第一时间知道 CLI 是否装好了。

    踩坑四：子进程找不到 claude 命令

    终端里 claude 能跑，插件里 spawn 就报 ENOENT。

    原因：uTools 启动的子进程继承的是它自己的环境变量，和用户 zsh/bash 配置文件里改过的 PATH 可能不一样。尤其是 npm 全局安装的工具，路径经常不在系统默认 PATH 里。

    解决：spawn 时传 shell: true，让 shell 去解析命令。同时前端顶部加状态栏，绿色表示检测通过，红色显示具体错误信息。

    踩坑五：工作目录硬编码

    一开始把默认 cwd 写死成了本地 vault 路径，结果代码里带了个人目录信息，换台机器直接用不了。

    优化后：

    • getCwd() 未配置时返回 null
    • 启动前检查，缺失直接报错：未设置工作目录。请在插件界面输入工作目录路径并保存。
    • UI 上加了个输入框 + 保存按钮，持久化到 utools.db

    UI 交互细节

    几个小点：

    Enter 发送：输入框监听 keydown，e.key === 'Enter' 时触发发送。

    再问一个：结果区域加了个”再问一个”按钮，点击后重置状态，显示输入框并自动聚焦，不需要重启插件。

    复制结果：优先用 utools.copyText(text)， fallback 到 navigator.clipboard。

    关闭窗口：调用 utools.outPlugin()。

    安全注意

    • 不硬编码个人目录路径。
    • 敏感路径必须报错提示用户手动配置，不要静默使用默认值。
    • shell: true 配合用户输入要小心命令注入。这个插件里没有拼接外部输入到命令里，风险可控。

    后续可优化方向

    • 流式输出：逐字显示回答，而不是等进程结束一次性展示。需要把 child.stdout.on('data') 实时推送到 renderer。
    • 伪终端支持：用 node-pty 启动 Claude Code，彻底解决 TUI 依赖问题（但会引入原生依赖，打包麻烦）。
    • 历史记录：把问答记录存到 db 里，支持翻查过往对话。

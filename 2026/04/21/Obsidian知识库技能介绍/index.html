    从零开始：为 Claude Code 打造一个 Obsidian 知识库写入技能

    概述

    Claude Code 是个好东西，但对话结束后那些有价值的内容就散落在历史记录里了。想着能不能让 Claude 直接把内容写到 Obsidian 里，这样既能保留上下文，又能利用 Obsidian 的知识管理能力。

    折腾了一下午，搞出来一个 Skill，支持三种场景：备忘录、对话总结、知识理解。这篇文章记录一下实现思路和踩过的坑。

    为什么要做这件事

    用 Claude Code 的过程中经常遇到这些情况：

    • 查了半天资料，想记个笔记，得手动复制粘贴到 Obsidian
    • 聊完一个技术话题，想总结一下，又得新开窗口整理
    • 看到一篇好文章让 Claude 解读，解读完内容就丢了

    理想情况是：直接告诉 Claude “记录一下”、”总结这段对话”、”理解这篇文章”，它自动把内容整理成 Obsidian 文档，还带上标签和时间戳。

    设计思路

    三种文档类型

    根据使用场景，划分了三种类型：

    类型 目录 触发词 标签
    memo memo/ “记录一下”、”备忘” #memo
    repository repository/ “总结对话”、”归纳一下” #repository
    knowledge knowledge/ “理解这篇文章” #knowledge

    memo 是随手记，想到什么记什么。repository 是对话的沉淀，把讨论过程整理成可检索的知识。knowledge 是对外部内容的理解，比如文章、GitHub 仓库。

    文档格式

    所有文档都用 Obsidian 原生的 Markdown + YAML frontmatter：

    正文分两部分：原始内容 + AI 的理解总结。这样既能保留原始信息，又有 AI 的加工提炼。

    技术实现

    Skill 结构

    SKILL.md 是给 Claude 看的”使用说明书”，定义了触发条件、操作流程、文件名规则。

    文件名生成

    一开始想省事，用时间戳命名：memo-20240115-093000.md。但这样根本看不出文件内容，目录一多就懵了。

    后来改成有意义的命名：memo-{关键词摘要}-{YYYYMMDD}.md

    比如：

    • memo-字节-OpenClaw-安装实践-20260421.md
    • know-Agent超级进化-ByteTech-20260421.md

    关键词从标题或内容里提取，去掉”的、了、and、the”这些虚词。长度控制在 80 字符以内，太长 Obsidian 显示不全。

    Claude 提取文章信息，生成：

    内容包括文章摘要、AI 的理解、待办事项。

    总结对话

    聊完一个话题：

    “总结这段对话”

    生成 repository 文档，包含讨论主题、关键决策、待办事项。

    理解 GitHub 仓库

    看到个有趣的仓库：

    “记录备忘 https://github.com/kepano/obsidian-skills“

    Claude 分析仓库 README，提取功能列表，给出借鉴建议。

    踩过的坑

    文件路径问题

    一开始 SKILL.md 里写的路径是相对于 vault 根目录的，但 Claude 执行时工作目录可能不对。后来改成先检查目录是否存在，不存在就创建。

    文件名长度

    中文文件名在 Git 里显示正常，但终端有时候会有编码问题。好在 Obsidian 和 Git 都支持 Unicode，实际使用没问题。

    标签规范

    Obsidian 的标签不能有空格，必须写成 #ai工具 而不是 #ai 工具。SKILL.md 里要明确这一点。

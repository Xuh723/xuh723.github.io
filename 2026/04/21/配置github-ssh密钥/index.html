    配置 GitHub SSH 密钥

    概述

    通过 SSH 密钥连接 GitHub 比每次输入用户名密码要方便得多，而且安全性也更高。本文记录一下使用 ed25519 算法生成密钥，并配置到 GitHub 上的完整流程。

    ed25519 相比传统的 RSA 算法，密钥更短、生成速度更快，而且安全性也足够。GitHub 从 2021 年底开始已经支持 ed25519，新配置的密钥建议优先选择这个算法。

    生成 SSH 密钥

    首先检查本地是否已经有 SSH 密钥。默认情况下，SSH 密钥存放在 ~/.ssh 目录下。

    如果已经存在 id_ed25519 和 id_ed25519.pub 文件，且确认是给 GitHub 使用的，可以直接跳到下一步配置。如果想重新生成，备份后删除原有文件即可。

    接下来会提示设置 passphrase（密钥密码）。这个密码是在使用密钥时需要输入的，可以为空，但为了安全建议设置一个。后续可以通过 ssh-agent 来避免重复输入密码。

    生成完成后，可以用 ls ~/.ssh 查看，应该能看到两个文件：

    • id_ed25519：私钥，绝对不能泄露
    • id_ed25519.pub：公钥，需要配置到 GitHub 上

    然后去 GitHub 网站添加公钥：

    1. 点击右上角头像，选择 Settings
    2. 左侧菜单选择 SSH and GPG keys
    3. 点击 New SSH key 按钮
    4. Title 随便填，比如 “MacBook Pro” 或 “公司电脑”
    5. Key type 保持默认的 Authentication Key
    6. 把刚才复制的公钥内容粘贴到 Key 文本框中
    7. 点击 Add SSH key

    如果提示输入密码，输入 GitHub 账号密码确认即可。

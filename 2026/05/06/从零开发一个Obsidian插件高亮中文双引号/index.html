    第三步：支持自定义颜色——从硬编码到可配置

    需求变化

    硬编码 #66ccff 不够灵活，希望用户在设置面板里自行选择颜色。

    实现方案

    如果直接修改 CSS 文件，用户每次换颜色都要手动改代码，体验很差。更好的方案是：

    1. 用 CSS 变量作为颜色值：var(--chinese-quote-color, #66ccff)
    2. 用 Obsidian Setting API 做一个颜色选择器
    3. 当用户修改颜色时，通过 JS 更新 CSS 变量，编辑器实时生效，无需重载

    启用插件

    1. 打开 Obsidian，进入 Settings → Community plugins
    2. 关闭 Safe mode
    3. 在已安装插件列表中找到 Chinese Quote Highlight，点击开启
    4. 点击 Options 进入设置面板，用 Color Picker 选择你喜欢的颜色

    修改颜色后编辑器中的中文引号会立即变色，无需重载应用。

    总结

    整个开发流程走下来，核心其实就三点：

    1. CodeMirror 扩展负责在编辑器中识别并装饰目标文本；
    2. CSS 变量 + JS 动态设置让颜色可配置且实时生效；
    3. Obsidian Setting API 提供原生设置面板，用户体验最佳。

    过程中踩的两个比较深的坑——manifest.json 的 JSON 语法错误和模块循环依赖——都是那种”代码看着没问题，但就是不生效”的问题。记录在这里，希望对你有所帮助。
